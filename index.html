Process

How PhishGuard Works

From URL submission to threat verdict in milliseconds

01

URL Submission & Preprocessing

Submit any URL through our web interface or REST API. The system normalizes the URL, extracts components, and prepares it for multi-layer analysis including domain parsing, path analysis, and query parameter inspection.

02

NLP Heuristic Analysis

25+ linguistic heuristics analyze the URL for typosquatting, brand impersonation, homograph attacks, suspicious TLD usage, subdomain patterns, and lexical features. Each heuristic produces a risk score.

03

ML Ensemble Scoring

Four machine learning models analyze the extracted features simultaneously. Random Forest and XGBoost handle tabular features while LSTM and BERT process sequential and semantic patterns. Results are weighted and combined.

04

Verdict & Reporting

The ensemble produces a final confidence score and threat classification (Safe / Suspicious / Phishing). Results are logged, the user is notified, and the analysis is stored for future reference and model improvement.
